## Erreurs rencontrees

### tchap-reader PVC Multi-Attach
- **Symptome** : Le nouveau pod tchap-reader restait en `ContainerCreating` apres `kubectl set image`
- **Cause** : Le PVC `tchap-reader-data` (ReadWriteOnce) etait attache au noeud de l'ancien pod. Le nouveau pod a ete schedule sur un autre noeud.
- **Fix** : Suppression manuelle de l'ancien pod (`kubectl delete pod`) pour liberer le PVC. Le nouveau pod a demarre ensuite.
- **Recommendation** : Utiliser `strategy.type: Recreate` au lieu de `RollingUpdate` pour les deployments avec PVC RWO.

### tchap-reader health (6d)
- **Symptome** : `status: unhealthy`, missing config `TCHAP_ACCESS_TOKEN`, `TCHAP_USER_ID`, `TCHAP_ALLOWED_ROOM_IDS`
- **Cause** : Credentials Tchap (Matrix) non configurees dans le ConfigMap. Le service HTTP fonctionne mais ne peut pas se connecter a Tchap.
- **Impact** : Les fonctions `tchap_connect`, `tchap_search_rooms`, etc. ne sont pas operationnelles. Non bloquant pour les autres tools.

### searxng pod CrashLoopBackOff
- **Symptome** : Pod `searxng-5cf498fb4b-p7t6t` en CrashLoopBackOff (224 restarts)
- **Cause** : Ancien ReplicaSet avec une config obsolete. Le pod actif (`searxng-579fcc8c89-55vb4`) fonctionne correctement.
- **Recommendation** : Nettoyer l'ancien ReplicaSet.
